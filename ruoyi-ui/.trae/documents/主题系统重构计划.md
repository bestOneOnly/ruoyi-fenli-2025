# 主题系统重构计划

## 1. 需求分析
- 取消三元表达式判断，改为使用当前主题类
- 将sidebar.scss主题拆分，使用三个文件代表三个主题

## 2. 实现方案

### 2.1 创建主题文件结构
- 创建 `src/assets/styles/themes` 目录
- 创建三个主题文件：
  - `theme-dark.scss` - 深色主题样式
  - `theme-light.scss` - 浅色主题样式  
  - `theme-green.scss` - 绿色主题样式

### 2.2 拆分主题样式
- 将 `sidebar.scss` 中的主题相关样式拆分到对应主题文件
- `sidebar.scss` 只保留通用样式，导入三个主题文件

### 2.3 修改组件实现
- **Sidebar/index.vue**：移除三元表达式，使用主题类控制样式
- **Logo.vue**：移除三元表达式，使用主题类控制样式
- 利用已有的 `settings.sideTheme` 类名（已应用到 `el-scrollbar`）

### 2.4 主题样式设计
- 每个主题文件包含完整的主题样式定义
- 使用CSS选择器优先级确保主题样式正确应用
- 保留现有的绿色主题设计（渐变背景、圆角边框等）

## 3. 预期效果
- 主题切换功能保持不变
- 代码结构更清晰，便于维护和扩展
- 移除冗余的三元表达式，提高代码可读性
- 每个主题样式独立管理，便于后续修改和添加新主题

## 4. 实施步骤
1. 创建主题目录和文件
2. 拆分 `sidebar.scss` 中的主题样式
3. 修改 `sidebar.scss` 导入主题文件
4. 修改 `Sidebar/index.vue` 移除三元表达式
5. 修改 `Logo.vue` 移除三元表达式
6. 测试主题切换功能

## 5. 技术要点
- 利用CSS类名优先级实现主题切换
- 保持现有的主题切换逻辑不变
- 确保所有主题样式正确应用
- 保持代码的可维护性和扩展性
